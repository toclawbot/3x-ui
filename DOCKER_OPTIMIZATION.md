# Docker 构建优化记录

## 优化内容

### 1. .dockerignore 文件
- 忽略不必要的构建文件，减少构建上下文
- 保留必要的构建文件和脚本

### 2. Dockerfile 优化
- **依赖缓存分层**: 先复制 go.mod/go.sum，再运行 go mod download
- **层合并**: 合并 RUN 命令减少镜像层数
- **注释优化**: 增加清晰的分段注释

### 3. 构建效率提升
- **缓存命中率**: 依赖文件单独复制，避免代码变更时重新下载依赖
- **构建速度**: 减少不必要的文件传输

## 功能保持
- ✅ 多阶段构建架构不变
- ✅ fail2ban 配置完整
- ✅ 二进制文件和脚本权限正确
- ✅ 时区和环境变量配置完整
