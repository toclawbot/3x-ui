# Docker 构建优化记录

## 优化内容

### 1. .dockerignore 文件
- 忽略不必要的构建文件，减少构建上下文
- 保留必要的构建文件和脚本

### 2. Dockerfile 优化
- **依赖缓存分层**: 先复制 go.mod/go.sum，再运行 go mod download
- **层合并**: 合并 RUN 命令减少镜像层数
- **注释优化**: 增加清晰的分段注释
- **fail2ban 移除**: 简化镜像，移除不必要的安全依赖
- **路径修复**: 解决文件复制路径问题

### 3. 构建效率提升
- **缓存命中率**: 依赖文件单独复制，避免代码变更时重新下载依赖
- **构建速度**: 减少不必要的文件传输
- **镜像轻量化**: 移除 fail2ban 及相关依赖包

### 4. GitHub Actions Workflow 优化
- **标签策略**: 使用语义化版本标签（v2.9.3, v2.9, v2, latest）
- **变量对齐**: 统一使用 ${{ secrets.DOCKER_USERNAME }} 格式
- **缓存优化**: 添加 Go 模块缓存步骤
